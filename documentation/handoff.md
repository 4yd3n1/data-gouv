## What This Is

French civic intelligence platform. Next.js 16 App Router + Prisma 7 + PostgreSQL 14. Ingests open data from data.gouv.fr, INSEE, HATVP, Sénat, Assemblée Nationale. Goal: **maximum transparency through public data**. "Intelligence Bureau" aesthetic (dark, teal accents, Instrument Serif headings).

## Critical Quirks

### Schema field traps (wrong = TypeScript build errors)
- `{ prisma }` is a **named** export — `import prisma from "@/lib/db"` = build error
- `pnpm db:generate` required after every migration (build fails with "Property X doesn't exist on PrismaClient")
- `Scrutin.votes` (relation), `Scrutin.dateScrutin`, `Scrutin.sortCode`, `Scrutin.abstentions`
- `GroupeVote.abstentions` — no `nomOrgane` field (use `organeRef`)
- `Depute.groupe` (NOT `groupePolitique`); `Depute.departementRefCode` for joins (NOT `departementCode` — that can be `"099"`)
- `Lobbyiste.nom`, `Lobbyiste.categorieActivite`
- `Elu.nom`/`Elu.prenom`, `Elu.codeDepartement`
- `DensiteMedicale.specialite` (NOT `profession`)
- `EvenementJudiciaire` has no `createdAt`/`updatedAt` columns
- `ElectionLegislative`: no FK to `Departement` — overseas codes (`"ZZ"`) don't exist in COG; `codeDepartement` is plain string

### Operational state
- **Node 20.19.2+** required (Prisma 7 WASM, `.nvmrc` pins it, `pnpm dev` auto-switches)
- `ingest:elus` needs 8GB heap (`NODE_OPTIONS=--max-old-space-size=8192`)
- ARM communes (Paris/Lyon/Marseille): `postal-resolver.ts` replaces ARM codes with parent COM via `comparent`. Always filter `typecom: "COM"` for counts
- `ConflictSignal` is empty until `pnpm compute:conflicts` runs (UI has graceful fallbacks)
- `search_index` view requires `pnpm refresh:search` after route changes (canonical `/profils/*` + `/votes/scrutins/*` URLs since Session 43)
- Deputy/Senator photo URLs return 404 (AN changed URL structure); Avatar correctly falls back to initials
- `lucide-react` is intentionally **NOT** installed — use inline SVG everywhere

### HATVP pipeline
- **Name normalization mandatory** (Session 46): all cross-dataset name joins MUST use `nomNormalise` + `prenomNormalise` (compound btree-indexed on `Depute`, `Senateur`, `DeclarationInteret`, `PersonnalitePublique`). Source of truth: [`src/lib/normalize-name.ts`](../src/lib/normalize-name.ts) (NFD + strip U+0300-U+036F + lowercase + trim). Ingestion scripts populate on write; backfill via `pnpm tsx scripts/backfill-normalized-names.ts`. **NEVER** `{ equals, mode: "insensitive" }` — handles case but not accents.
- **HATVP merged.xml** is gitignored (~120MB), cached at `documentation/hatvp-old-context/declarations.xml`. HATVP server is flaky — re-download with `curl -C - --retry 30 --retry-delay 3 --retry-all-errors -o <path> https://hatvp.fr/livraison/merge/declarations.xml`. `ingest-hatvp.ts` prefers the local cache.
- HATVP XML parser: `fast-xml-parser` (NOT xml2js). Stream — files can be 200MB+
- 8 of 37 Lecornu II ministers genuinely haven't filed HATVP (Macron + 7 secretaries)
